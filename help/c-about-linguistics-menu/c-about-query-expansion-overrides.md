# クエリーの拡張の上書きについて{#about-query-expansion-overrides}

検索クエリの結果の展開を上書きできます。

## クエリーの拡張の上書きの使用 {#concept_6895B469B0E044299E93361BFA06B554}

クエリの拡張の上書きを設定する場合は、「ルール」のセットを作成します。 各ルールには、基本的に「検索時に拡張しない」と記述されています。この場合、は単純なテキスト単語またはフレーズ `<this>` で、はテキスト単語また `<that>` はフレーズ、または分類 `<this>``<that>` です。

>[!NOTE]
>
>この機能は、Search&amp;Promoteではデフォルトでは有効になっていません。 お使いの機能を有効にするには、テクニカルサポートにお問い合わせください。 クエリーの拡張の上書き機能を有効にした後、ユーザーインターフェイスでオンにする必要があります。

**クエリー拡張の上書きの仕組み**

「クエリの拡張による追加」ページで「テキスト」と「キーワード」の値が指定されている場合、コードは特定のペアに対して動作します。 辞書や代替単語フォームなど、分類タイプをキーワードとして指定した場合、「展開しない」の値は、指定した分類で作成されたフォームに変換されません。

例えば、次の定義があるとします。

`Do Not Expand = "dog"`

`Type = Text`

`Term = "dogs"`

「dog」の検索クエリは、「dog&#39;s」と「dogs」を含むように拡張され、「dogs」は代替単語フォームを使用して含まれません。

ただし、定義が次の場合は、

`Do Not Expand = "dog"`

`Type = Alternate Word Forms`

クエリに「dog&#39;s」または「dogs」（「dog」に対して使用可能な代替単語フォーム）が含まれていません。

複数のキーワード、複数の分類またはその両方を指定できます。 ただし、「種類」で「すべて」を選択した場合、複数用語のリストは1つの「すべて」のエントリに折りたたまれます。

テキストと分類のエントリがルール内で混在している場合は、最初にテキスト値が表示されるように、ユーザーインターフェイス内で再編成されます。 ただし、これは検索時の評価の順序を示すものでも、影響を与えるものでもありません。

テキスト用語は、意味のない参照を削除するために検証されます。 つまり、キーワードと「展開しない」の値を比較し、一致が見つかった場合はそのキーワードを削除します。 また、重複したキーワード値（テキストまたは分類）は削除されます。

以前の定義を複製する「展開しない」の値を持つ新しいルールを追加すると、新しい定義の用語が元の定義に追加されます。
