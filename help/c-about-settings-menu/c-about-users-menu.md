# [ユーザ]メニュー{#about-the-users-menu}

ユーザーメニューを使用して、ユーザーの表示と追加、ロールの表示と追加、またはロールのメンバーシップの変更を行います。 [ユーザ]メニューの任意のタスクを実行するには、管理者権限を持つSite Search/Merchandisingユーザである必要があります。

## アカウントユーザーの表示{#task_FDDF30EE23C548DF8CFBB2FB2605303C}

[!DNL View Users]ページを使用して、既存のすべてのアカウントユーザーを表示できます。 アカウントユーザー（アカウント所有者を除く）を削除することもできます。

**[!UICONTROL User Admin]**&#x200B;がオンになっているアカウントユーザーのみが、ユーザーを削除したり、ユーザーの役割を変更したりできます。

アカウント所有者を削除する前に、アカウント所有者を別のユーザーに転送します。

所有権を移行した後は、他のユーザーと同様にアカウントを削除できます。 ステータスの変更を通知する電子メールを受信しなくなりました。

「[アカウント所有権の別のアカウントユーザーへの移行](../c-about-settings-menu/c-about-account-options-menu.md#task_47E3C66CC6374274830DA10171E0CF17)」を参照してください。

>[!NOTE]
>
>このタスクを実行するには、Site Search/Merchandisingの管理者権限が必要です。

**表示アカウントユーザーに**

1. 製品メニューで、**[!UICONTROL Settings]**/**[!UICONTROL Users]**/**[!UICONTROL View Users]**&#x200B;をクリックします。
1. （オプション）「[!DNL User Admin]」列見出しの下で、アカウントユーザーの削除またはアカウントユーザーの役割の編集を許可するアカウントユーザーを選択します。
1. （オプション）「[!DNL Remove?]」列見出しの下で、削除するアカウントユーザーを選択します。
1. クリック **[!UICONTROL Save Changes]**.
1. （オプション）[!DNL View Users]ページのハイパーリンクロール名をクリックします。 [!DNL Role Membership]ページが開き、ユーザーをロールに割り当てたり、ユーザーのロールの割り当てを解除したりできます。

   [ロールのメンバーシップの設定](../c-about-settings-menu/c-about-users-menu.md#task_DAC596AAFFCE4EF0BE68CEEF7E365414)を参照してください。

## アカウントユーザーの追加{#task_176C463A0C0849B29245C28EC9876326}

[!DNL Add Users]ページを使用して、新しいアカウントユーザーをサイト検索/マーチャンダイジングに追加できます。

新しいユーザーの電子メールアドレスのみが必要です。 新しいユーザがアカウントに追加されると、新しいユーザにアカウント情報が送信されます。

デフォルトでは、新しいユーザーはユーザー管理者として割り当てられません。 ユーザー管理者は、ロールの定義と編集、他のユーザーの削除を実行できます。 [!DNL Add Users]ページから、新しいユーザーをユーザー管理者にすることを選択できます。 または、表示ユーザーページを使用して、新しいユーザーをユーザー管理者にすることができます。

「[アカウントユーザーの表示](../c-about-settings-menu/c-about-users-menu.md#task_FDDF30EE23C548DF8CFBB2FB2605303C)」を参照してください。

指定する電子メールアドレスには、ASCII文字のみを使用する必要があります。 標準の英字(a..z)文字または数値(0.9)文字。ユーザー名とドメインを区切るために使用される`@`文字は1つだけです。 `_`、`+`、`-`、`.`、`!`、`#`、`$`、`'`、`%`、`&`、`*`、`?`、`^`、a14/>と`}`も使用できます。 `=``{`電子メールアドレスを`.`に開始しない

新しいユーザーが既にAdobeのお客様でない場合は、そのユーザーのお客様ログインを作成するように求められます。 新しいユーザーに、ログインパスワードと確認が送信されます。 新規ユーザーは、初めてログインしたときに、顧客プロファイルを入力します。

必要に応じて、[!DNL Add Users]ページのハイパーリンクロール名をクリックできます。 [!DNL Role Membership]ページが開き、ユーザーをロールに割り当てたり、ユーザーのロールの割り当てを解除したりできます。

[ロールのメンバーシップの設定](../c-about-settings-menu/c-about-users-menu.md#task_DAC596AAFFCE4EF0BE68CEEF7E365414)を参照してください。

>[!NOTE]
>
>このタスクを実行するには、Site Search/Merchandisingの管理者権限が必要です。

**アカウントユーザーを追加するには**

1. 製品メニューで、**[!UICONTROL Settings]**/**[!UICONTROL Users]**/**[!UICONTROL Add Users]**&#x200B;をクリックします。
1. [!DNL Add Users]ページの[!DNL User's Email]フィールドと[!DNL User's Email (again)]フィールドに、新しいユーザーの電子メールアドレスを入力します。
1. （オプション）新しいアカウントユーザーに、アカウントユーザーの削除やアカウントユーザーの役割の編集を許可するには、**[!UICONTROL User Administrator]**&#x200B;をチェックします。
1. [!DNL Roles for this User]テーブルで、新しいアカウントユーザーに割り当てるロールを確認します。
1. クリック **[!UICONTROL Add User]**.

## アカウント{#task_4EAE1D018F384691A083AD51E5CE58DC}に存在するロールの表示

[!DNL View Roles]ページを使用して、現在ログインしているアカウントに現在存在するすべてのロールを表示できます。

アカウントにロールが存在しない場合は、ユーザーインターフェイスからこの問題が通知されます。 ロールを追加使用して、ロールを作成および追加できます。

「[アカウントへの新しいロールの追加](../c-about-settings-menu/c-about-users-menu.md#task_E148D02275DE4F899BA79736A29895AB)」を参照してください。

ロールを削除しても、ロールに割り当てられているアカウントユーザーは削除されません。

>[!NOTE]
>
>このタスクを実行するには、Site Search/Merchandisingの管理者権限が必要です。

**アカウントに存在するロールを表示するには**

1. 製品メニューで、**[!UICONTROL Settings]**/**[!UICONTROL Users]**/**[!UICONTROL View Roles]**&#x200B;をクリックします。
1. （オプション）テーブルの[!DNL Remove?]列ヘッダーの下で、削除するロールを選択します。
1. クリック **[!UICONTROL Save Changes]**.
1. （オプション）次のいずれかの操作を行います。

   * 表の[!DNL Role Name]列見出しの下で、ハイパーリンクされたロール名をクリックします。 [!DNL Role Membership]ページが開き、ユーザーをロールに割り当てたり、ユーザーのロールの割り当てを解除したりできます。

      [ロールのメンバーシップの設定](../c-about-settings-menu/c-about-users-menu.md#task_DAC596AAFFCE4EF0BE68CEEF7E365414)を参照してください。

   * テーブルの[!DNL Edit]列ヘッダーの下で、鉛筆アイコンをクリックします。 [!DNL Edit Role]ページが開き、ロールの名前の変更、説明の変更などを行うことができます。

      「[ロールの編集](../c-about-settings-menu/c-about-users-menu.md#task_13875C2464034CE387285640412E1B59)」を参照してください。

## ロールの編集{#task_13875C2464034CE387285640412E1B59}

既存のロールの名前を変更したり、説明を変更したり、ロールにアクセス権を与えるユーザーインターフェイスの領域のみを選択したりできます。

「[アカウントへの新しいロールの追加](../c-about-settings-menu/c-about-users-menu.md#task_E148D02275DE4F899BA79736A29895AB)」を参照してください。

>[!NOTE]
>
>このタスクを実行するには、Site Search/Merchandisingの管理者権限が必要です。

**ロールを編集するには**

1. 製品メニューで、**[!UICONTROL Settings]**/**[!UICONTROL Users]**/**[!UICONTROL View Roles]**&#x200B;をクリックします。
1. 表の[!DNL Edit]列ヘッダーの下で、変更するロールの横にある鉛筆アイコンをクリックします。
1. （オプション）次のいずれかの操作を行います。

   * 必要に応じて、「[!DNL Role Name]」テキストフィールドに新しい名前を入力します。 *は、このフィールドがロールに必須であることを示します。
   * 必要に応じて、[!DNL Description]テキストフィールドに役割の新しい説明を入力します。
   * グループボックスで、ロールにアクセスさせる機能またはブロックする機能をそれぞれオンまたはオフにします。

1. クリック **[!UICONTROL Save Changes]**.

## アカウントに新しいロールを追加する{#task_E148D02275DE4F899BA79736A29895AB}

[!DNL Add Roles]ページを使用すると、アカウントユーザーへの権限の割り当てを簡単かつ迅速に行うことができます。

例えば、広報部門の各メンバーに、サイト検索/マーチャンダイジングタスクへのアクセスを個別に付与できます。 ただし、これらのユーザーを「PR」ロールに追加し、タスクをロール全体に割り当てる方がはるかに効率的です。

各ロール名は一意である必要があります。 ダッシュ`"-"`、アンダースコア`"_"`、ピリオド`"."`などの英数字と共通の記号を使用できます。 名前の先頭にアンダースコアまたはピリオドを付けることはできません。

表の[!DNL Users in This Role]列ヘッダーの下で、必要に応じて、ユーザーのハイパーリンク付き電子メールアドレスをクリックできます。 [!DNL Role Membership]ページが特定のユーザーに対して開きます。このユーザーにロールを割り当てるか、ユーザーのロールの割り当てを解除することができます。

[ロールのメンバーシップの設定](../c-about-settings-menu/c-about-users-menu.md#task_DAC596AAFFCE4EF0BE68CEEF7E365414)を参照してください。

表の[!DNL Roles]列見出しの下で、必要に応じて、ハイパーリンクロール名をクリックできます。 [!DNL Role Membership]ページが開き、選択したロールにユーザーを割り当てたり、選択したロールからユーザーの割り当てを解除したりできます。

>[!NOTE]
>
>このタスクを実行するには、Site Search/Merchandisingの管理者権限が必要です。

**アカウントに新しいロールを追加するには**

1. 製品メニューで、**[!UICONTROL Settings]**/**[!UICONTROL Users]**/**[!UICONTROL Add Roles]**&#x200B;をクリックします。
1. [!DNL Add Roles]ページの[!DNL Role Name]フィールドにロールの名前を入力します。
1. （オプション）「[!DNL Description]」フィールドに、役割を適切に説明した文を入力します。
1. 各電子メールアドレスの左側のボックスをオンにして、役割に属するユーザを選択します。
1. クリック **[!UICONTROL Add Role]**.

## ロールメンバーシップの構成{#task_DAC596AAFFCE4EF0BE68CEEF7E365414}

また、ユーザーごとにロールを表示する場合は、個々のユーザーグループメンバーシップを管理することもできます。

「[アカウントへの新しいロールの追加](../c-about-settings-menu/c-about-users-menu.md#task_E148D02275DE4F899BA79736A29895AB)」を参照してください。
